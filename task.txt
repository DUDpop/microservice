Разработать микросервис для получение данных из справочников различной структуры, используя PostgreSQL в качестве базы данных и Spring Boot для реализации сервиса.

Описание задачи:
1. Структура справочников:
- Справочники могут содержать различные типы данных (например, строки, числа, даты и т.д.).
- Каждый справочник имеет свою структуру, которая может изменяться со временем (добавление новых полей, удаление старых).
2. Функциональные требования:
- Получение списка всех доступных справочников.
- Добавление, обновление и удаление записей в справочнике.
- Получение записей из справочника по ключевому полю.
3. Технические требования:
- Использовать PostgreSQL для хранения данных.
- Использовать Spring Boot для разработки микросервиса.
- Реализовать API для взаимодействия с сервисом (REST).
- Документация API с использованием Swagger или OpenAPI.
4. Дополнительные требования:
- Реализовать создание структуры БД, ее изменение и первичное наполнение с использованием Liquibase (при запуске микросервиса проверять актуальность состояния БД).
- Обеспечить логирование операций (необходимое для мониторинга и отладки микросервиса).
- Написать unit-тесты для основных компонентов сервиса.
